# Быстрый старт

Подключаем AI-ассистента к вашей базе 1С за 10 минут.

## Что это и зачем

**Проблема:** Вы пишете код на BSL в 1С. Современные AI-модели (YandexGPT, GigaChat, Claude и др.) умеют генерировать код, но не знают структуру *вашей* конфигурации — какие справочники, документы, реквизиты у вас есть. Без этого AI выдумывает несуществующие реквизиты и пишет нерабочий код.

**Решение:** mcp-1c — мост между AI-ассистентом и вашей базой 1С. После установки AI видит:
- Все объекты конфигурации (справочники, документы, регистры, ...)
- Реквизиты, табличные части, измерения, ресурсы
- Код модулей (через поиск по выгрузке конфигурации)
- Структуру форм
- Данные через запросы (только SELECT)

AI не просто «знает 1С вообще» — он знает *именно вашу* конфигурацию и пишет точный код под неё.

### Что такое MCP

MCP (Model Context Protocol) — открытый протокол, который позволяет AI-ассистентам обращаться к внешним источникам данных. Это стандарт, который поддерживают десятки AI-клиентов: [Claude Desktop](https://claude.ai/download), [Cursor](https://cursor.com), [Windsurf](https://codeium.com/windsurf) и другие. Полный список — на [modelcontextprotocol.io](https://modelcontextprotocol.io).

Вам не нужно разбираться в деталях протокола — просто установите mcp-1c и AI-ассистент получит доступ к вашей базе.

### Как это работает

```
┌─────────────────┐         ┌──────────────┐         ┌──────────────┐
│  AI-ассистент   │  stdio  │  mcp-1c.exe  │  HTTP   │  База 1С     │
│  (любой MCP-    │ ──────► │  (посредник)  │ ──────► │  (расширение │
│   совместимый)  │ ◄────── │              │ ◄────── │   HTTP-сервис)│
└─────────────────┘         └──────────────┘         └──────────────┘
```

1. Вы задаёте вопрос AI про вашу конфигурацию
2. AI через mcp-1c запрашивает нужные метаданные из 1С
3. Получает реальную структуру и пишет код, который подходит именно вашей базе

## Требования

- **1С:Предприятие 8.3 или 8.5** (коммерческая или учебная версия)
- **AI-клиент с поддержкой MCP** — например [Claude Desktop](https://claude.ai/download) или [Cursor](https://cursor.com)
- **Windows** или **Linux** (на macOS HTTP-сервис 1С [не поддерживается](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/), но mcp-1c и автоустановка работают)

> Учебную версию 1С можно скачать бесплатно: [uc1.1c.ru](https://uc1.1c.ru/uchebnaya-versiya-1s/)

## Установка

### Шаг 1. Скачать mcp-1c

Скачайте `mcp-1c.exe` (Windows) или `mcp-1c` (macOS/Linux) со страницы [Releases](../../releases).

> Или соберите из исходников: `go build -o mcp-1c ./cmd/mcp-1c/`

### Шаг 2. Установить расширение в базу 1С

Расширение добавляет в вашу базу HTTP-сервис, через который mcp-1c получает данные. Оно **не меняет** вашу конфигурацию — это именно расширение.

**Автоматически (рекомендуется):**

```bash
# Windows
mcp-1c --install "C:\путь\к\вашей\базе"

# macOS / Linux
mcp-1c --install ~/Documents/InfoBase
```

Программа сама найдёт установленную платформу 1С, загрузит расширение и обновит конфигурацию БД.

> Если платформа установлена в нестандартную папку, укажите путь явно:
> `mcp-1c --install "путь_к_базе" --platform "/custom/path/to/1cv8"`

> Перед установкой закройте все сеансы 1С (Конфигуратор, Предприятие) для этой базы.

**Вручную (если автоматическая не сработала):**

1. Откройте базу в **Конфигураторе**
2. Меню **Конфигурация → Расширения конфигурации**
3. Нажмите **«Добавить из файла»** и выберите `extension/MCP_HTTPService.cfe` из скачанного архива
4. Нажмите **F7** (обновить конфигурацию базы данных)

### Шаг 3. Запустить HTTP-сервис 1С

Чтобы mcp-1c мог обращаться к базе, нужно запустить встроенный HTTP-сервер 1С.

**Windows (коммерческая версия):**
```cmd
"C:\Program Files\1cv8\8.3.XX.XXXX\bin\1cv8.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

**Windows (учебная версия):**
```cmd
"C:\Program Files\1cv8t\8.5.XX.XXXX\bin\1cv8t.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

> Замените `XX.XXXX` на вашу версию платформы. Посмотреть можно в папке `C:\Program Files\1cv8\` или `C:\Program Files\1cv8t\`.

**macOS:**

Платформа 1С [не поддерживает HTTP-сервисы на macOS](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/). Используйте Windows (можно в виртуальной машине, например UTM).

**Linux:**

Встроенный HTTP-сервер (`/HTTPPort`) недоступен. Используйте публикацию через Apache или автономный сервер ibsrv.

Окно 1С откроется и будет работать как HTTP-сервер. **Не закрывайте его** — пока оно открыто, HTTP-сервис доступен.

**Проверка — откройте в браузере или выполните в терминале:**

```
curl http://localhost:8080/hs/mcp/version
```

Ожидаемый ответ:
```json
{"version":"0.4.0"}
```

Если видите ответ — HTTP-сервис 1С работает, переходите к шагу 4.

### Шаг 4. Настроить AI-клиент

Ниже пример для Claude Desktop. Для других MCP-клиентов — смотрите их документацию.

Откройте файл конфигурации:

- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

Если файла нет — создайте его. Добавьте:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp"]
    }
  }
}
```

> Путь к `mcp-1c.exe` — полный путь к файлу, который вы скачали на шаге 1.
> Обратные слеши в JSON удваиваются: `C:\\Users\\...`.

**Перезапустите AI-клиент** (полностью закройте и откройте заново).

> **Claude Desktop: режим доступа инструментов.**
> По умолчанию Claude Desktop загружает инструменты «по запросу» (Auto) — при большом количестве подключённых серверов это экономит контекст, но может приводить к тому, что часть инструментов mcp-1c не обнаруживается при поиске. Если вы используете только mcp-1c (или 1–2 коннектора), рекомендуем переключить режим на **Always Available**:
>
> **«+» → Connectors → Tool access → Always available**
>
> В этом режиме все 9 инструментов загружаются сразу и AI видит их целиком.

### Шаг 5. Проверить

Напишите AI-ассистенту:

> Покажи структуру конфигурации моей базы 1С

AI вызовет инструмент `get_metadata_tree` и покажет список справочников, документов и регистров вашей базы.

## Примеры использования

После настройки AI может:

**Узнать о базе:**
> На какой конфигурации и платформе работает эта база?

**Анализировать конфигурацию:**
> Какие реквизиты есть у документа РеализацияТоваровУслуг?

**Писать код под вашу базу:**
> Напиши обработку, которая находит все документы РеализацияТоваровУслуг за последний месяц и выводит итог по сумме

**Работать с запросами (с параметрами):**
> Покажи все реализации контрагенту «ООО Ромашка» за март 2026

**Читать код модулей:**
> Покажи код модуля объекта справочника Контрагенты

**Искать по коду:**
> Найди все места, где используется функция ПолучитьОстаткиТоваров

**Анализировать ошибки:**
> Покажи ошибки из журнала регистрации за сегодня

**Справка по BSL:**
> Какой синтаксис у функции СтрНайти?

## Аутентификация

Если в базе 1С настроена аутентификация для HTTP-сервисов:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": [
        "--base", "http://localhost:8080/hs/mcp",
        "--user", "Admin",
        "--password", "пароль"
      ]
    }
  }
}
```

Или через переменные окружения:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp"],
      "env": {
        "MCP_1C_USER": "Admin",
        "MCP_1C_PASSWORD": "пароль"
      }
    }
  }
}
```

## Публикация через IIS / Apache (продвинутый вариант)

Встроенный HTTP-сервер подходит для разработки. Для постоянной работы опубликуйте HTTP-сервис через IIS или Apache:

1. Откройте базу в **Конфигураторе от администратора**
2. **Администрирование → Публикация на веб-сервере**
3. Укажите имя (например `base`), отметьте `MCPService`
4. Нажмите **Опубликовать**

URL в этом случае: `http://localhost/base/hs/mcp`

## Устранение неполадок

**«1C platform not found» при --install**
Платформа 1С не найдена в стандартных папках. Укажите путь явно: `mcp-1c --install "путь" --platform "/path/to/1cv8"`

**«Файл не найден» при --install**
Проверьте путь к базе. Это папка, в которой лежит файл `1Cv8.1CD`.

**«connection refused» при curl**
1С не запущена или не запущена с ключом `/HTTPPort`. Убедитесь, что окно 1С открыто.

**«404» при обращении к /hs/mcp/...**
- Расширение не установлено или не активно. Проверьте: Конфигуратор → Расширения → галочка «Активно» напротив MCP_HTTPService.
- Конфигурация БД не обновлена. Нажмите F7 в Конфигураторе.

**«address already in use»**
Порт 8080 занят. Используйте другой: `/HTTPPort 9090` и обновите URL в конфиге AI-клиента.

**AI не видит инструменты 1С**
- Убедитесь, что AI-клиент перезапущен после изменения конфига.
- Проверьте JSON на ошибки (лишние/недостающие запятые, неэкранированные слеши).
- Убедитесь, что путь к `mcp-1c.exe` правильный.

**AI не находит нужный инструмент (например get_object_structure)**
- В Claude Desktop: **«+» → Connectors → Tool access → Always available**. В режиме Auto (по умолчанию) Claude загружает только 5 инструментов за раз и может не найти нужный.

**«Достигнуто ограничение учебной версии» / «Предельное количество подключений»**
- Учебная версия 1С ограничивает количество одновременных подключений. Закройте лишние сеансы (Конфигуратор, другие окна 1С). mcp-1c закрывает HTTP-соединения после каждого запроса, чтобы минимизировать эту проблему.
